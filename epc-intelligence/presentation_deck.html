Technical Innovation Slide 4 / 7

Clause-Tree Retrieval vs Traditional Vector RAG

Traditional Vector RAG

  • • Documents split into arbitrary 500-token chunks
  • • Chunks cut sentences/lists halfway, destroying context
  • • Cosine similarity loses semantic relationships
  • • Hard to verify citations back to exact legal sections

AegisEPC Clause-Tree RAG

  • • Parser maps headers to a nested clause tree
  • • Preserves document hierarchy (Section → Clause X.Y)
  • • LLM navigates TOC, then extracts target clause text
  • • 100% deterministic, zero embeddings, zero vector DB cost
